# canopen-rs

CANopen protocol stack for embedded Rust. `no_std`, `no_alloc`, uses `embedded-can` 0.4 traits.

## Workspace

- `canopen-core/` — protocol implementation (NMT, SDO, PDO, heartbeat, EMCY, SYNC)
- `canopen-derive/` — proc macros (`object_dictionary!`, `object_dictionary_from_eds!`)
- `canopen-linux/` — socketcan transport + SDO client helpers for Linux
- `hil-tests/` — hardware-in-the-loop test harness
- `examples/stm32-node/` — Embassy firmware for STM32F767ZI (excluded from workspace, cross-compiled)

## Build & Test

```sh
cargo build                  # workspace (Linux side)
cargo test                   # 41 unit + integration tests

# STM32 firmware (separate project)
cd examples/stm32-node
cargo build --release
probe-rs run --chip STM32F767ZITx target/thumbv7em-none-eabihf/release/stm32-canopen-node

# HIL tests (requires CAN hardware or vcan)
CAN_IFACE=can0 cargo test -p hil-tests -- --test-threads=1 --ignored
```

## Architecture

- `Transport` trait abstracts CAN bus (polling-based). `MailboxTransport` for interrupt/async bridging.
- `ObjectDictionary` trait: generated by proc macro, maps index/subindex to typed struct fields.
- `Node<OD>` ties together NMT, SDO server, PDO engines, heartbeat. Call `node.process(&mut transport, &clock)` periodically.
- `SdoClient` is standalone for talking to remote nodes.
- All types use static allocation (heapless).
- STM32 firmware uses Embassy channels to bridge async CAN driver ↔ polling Node.

## OD Definition

Inline DSL:
```rust
object_dictionary! {
    pub struct MyOd {
        [0x1000] device_type: u32 = 0x191, ro;
        [0x6000] inputs: record {
            [1] input1: u8 = 0, ro, pdo;
        };
    }
}
```

From EDS file:
```rust
object_dictionary_from_eds! {
    pub struct MyOd = "device.eds";
}
```

## HIL Test Setup

- Nucleo-G431KB connected via ST-Link (probe-rs)
- ESP32 SLCAN adapter on USB (creates socketcan interface via slcand)
- Both on the same CAN bus at 500 kbps
- SLCAN setup:
  ```sh
  sudo slcand -o -c -s6 /dev/ttyUSB0 slcan0   # s6 = 500kbps
  sudo ip link set up slcan0
  ```
- Run HIL tests: `CAN_IFACE=slcan0 cargo test -p hil-tests -- --test-threads=1 --ignored`

## STM32 Notes

- Nucleo-G431KB: STM32G431KBTx
- embassy-stm32 v0.4 with `stm32g431kb` feature
- FDCAN1 peripheral in classic CAN mode, 500 kbps
- Pins: PA11 (RX) / PA12 (TX)
- Clock: HSI 16MHz → PLL → 170MHz (no external crystal on this Nucleo)
